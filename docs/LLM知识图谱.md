## 知识模块详解

### 模块一：基础理论（Why）

#### 1.1 Scaling Law（缩放定律）

**核心定义**：模型性能与规模（参数、数据、算力）呈幂律关系

**关键公式**：
```
L(N) = (N_c / N)^α

L: 损失（越低越好）
N: 模型参数量
N_c, α: 常数（α 约 0.07-0.35，取决于任务）
```

**重要认知**：
- 性能随规模平滑提升 → 可预测、可规划
- 临界点后会"涌现"新能力 → 量变引起质变
- 存在最优计算预算分配 → Chinchilla 最优（参数量 ∝ 数据量）

**参考资料**：Kaplan et al. (2020), Hoffmann et al. (2022, Chinchilla)

---

#### 1.2 涌现能力（Emergent Abilities）

**定义**：小模型不具备、大模型突然展现的能力

**典型涌现能力**：

| 能力 | 描述 | 典型规模 |
|------|------|----------|
| **上下文学习** | 通过 prompt 示例学习新任务 | ~10B+ |
| **指令遵循** | 理解并执行自然语言指令 | ~68B+ (FLAN) |
| **思维链推理** | 逐步分解解决复杂问题 | ~100B+ |
| **代码生成** | 根据描述生成可运行代码 | ~10B+ (Codex) |

**关键洞察**：涌现不是"魔法"，是统计规律在高维空间的体现

### 模块二：核心技术（How）

#### 2.1 Transformer 核心组件

```
输入文本 → Tokenize → Embedding → [多层 Transformer] → 输出概率

单层 Transformer 内部：
┌─────────────────────────────────────────────────────┐
│  Input X                                            │
│     │                                               │
│     ▼                                               │
│  LayerNorm                                          │
│     │                                               │
│     ▼                                               │
│  Multi-Head Attention ────────────────┐            │
│     │                                 │ 残差连接    │
│     ▼                                 │            │
│  X + Attention(X) ◄───────────────────┘            │
│     │                                               │
│     ▼                                               │
│  LayerNorm                                          │
│     │                                               │
│     ▼                                               │
│  Feed Forward (MLP) ────────────────┐              │
│     │                               │ 残差连接      │
│     ▼                               │              │
│  Output ◄───────────────────────────┘              │
└─────────────────────────────────────────────────────┘
```

**关键设计原理**：
1. **Self-Attention**：任意位置直接交互，解决长距离依赖
2. **残差连接**：缓解梯度消失，支持深层网络（100+层）
3. **LayerNorm**：稳定训练过程
4. **FFN**：增加非线性表达能力

---

#### 2.2 注意力机制深度解析

**计算流程**（矩阵视角）：
```
给定输入 X ∈ R^(n×d)

Q = X·W_Q    (Query)
K = X·W_K    (Key)  
V = X·W_V    (Value)

Attention(Q,K,V) = softmax(QK^T/√d_k)·V

输出 ∈ R^(n×d)，每个位置聚合了全局信息
```

**为什么除以 √d_k？**
- 防止点积过大导致 softmax 梯度消失
- 保持方差稳定（点积方差 ∝ d_k）

**因果掩码（Causal Mask）**：
```
对于位置 i，只能看 0~i 的位置

掩码矩阵（下三角）：
[[1, 0, 0, 0],
 [1, 1, 0, 0],
 [1, 1, 1, 0],
 [1, 1, 1, 1]]

被掩码位置填充 -inf，softmax 后为 0
```

#### 2.3 训练三阶段

```
┌────────────────────────────────────────────────────────────────┐
│ Stage 1: 预训练 (Pre-training)                                  │
├────────────────────────────────────────────────────────────────┤
│ 数据：万亿级 tokens（网页、书籍、代码）                          │
│ 任务：Next Token Prediction                                     │
│ 成本：数百万美元（GPT-3 约 $4.6M）                              │
│ 产出：基础模型（Base Model）                                     │
│ 能力：语言流畅，有知识，但不会对话                               │
└────────────────────────────────────────────────────────────────┘
                              ↓
┌────────────────────────────────────────────────────────────────┐
│ Stage 2: 监督微调 (SFT)                                         │
├────────────────────────────────────────────────────────────────┤
│ 数据：数十万高质量对话（Instruction-Output 对）                 │
│ 任务：监督学习（给定输入，预测期望输出）                        │
│ 成本：数千美元                                                   │
│ 产出：指令模型（Instruct Model）                                │
│ 能力：能对话，遵循指令，但可能有毒/偏见                         │
└────────────────────────────────────────────────────────────────┘
                              ↓
┌────────────────────────────────────────────────────────────────┐
│ Stage 3: RLHF 对齐                                              │
├────────────────────────────────────────────────────────────────┤
│ 数据：人类对多个回答的偏好排序                                   │
│ 任务：训练 Reward Model + PPO 强化学习                          │
│ 成本：数万美元（需要人工标注）                                   │
│ 产出：对齐模型（Aligned Model）                                 │
│ 能力：有用、无害、诚实                                           │
└────────────────────────────────────────────────────────────────┘
```

### 模块三：应用实践（What）

#### 3.1 RAG 系统架构

**为什么需要 RAG？**
- LLM 有知识截止日期
- LLM 可能"幻觉"（自信地编造）
- 企业有私有数据需要利用

**RAG 核心流程**：
```
用户提问
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 1. 检索 (Retrieval)                                          │
│    - 问题 → Embedding → 向量相似度搜索                        │
│    - 返回 Top-K 相关文档片段                                  │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 2. 增强 (Augmentation)                                       │
│    - 将检索结果注入 Prompt                                    │
│    - "基于以下信息回答问题：[文档片段]"                       │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────────────────┐
│ 3. 生成 (Generation)                                         │
│    - LLM 基于检索到的知识生成回答                             │
│    - 可溯源，减少幻觉                                         │
└──────────────────────────────────────────────────────────────┘
    │
    ▼
返回回答
```

**关键技术点**：
- **Embedding 模型选择**：OpenAI (通用)、BGE (中文)、M3E (轻量)
- **分块策略**：固定大小 vs 语义分块 vs 递归分块
- **向量数据库**：Chroma (原型)、Milvus (生产)、Pinecone (托管)
- **重排序（Rerank）**：精排提升检索质量

---

#### 3.2 Agent 框架

**什么是 Agent？**
> 能感知环境、做出决策、执行动作的智能体

**核心组件**：
```
┌────────────────────────────────────────────────────────────────┐
│                         Agent 架构                             │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐        │
│  │   Planning  │    │   Memory    │    │    Tools    │        │
│  │   规划模块   │    │   记忆模块   │    │   工具模块   │        │
│  └──────┬──────┘    └──────┬──────┘    └──────┬──────┘        │
│         │                  │                  │               │
│         └──────────────────┼──────────────────┘               │
│                            │                                  │
│                            ▼                                  │
│                     ┌─────────────┐                          │
│                     │     LLM     │                          │
│                     │   (大脑)    │                          │
│                     └─────────────┘                          │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

**Planning 方法**：
- **ReAct**：推理 + 行动交替进行（Thought → Action → Observation）
- **CoT**：思维链，逐步分解复杂任务
- **ToT**：思维树，探索多种解决方案

**Memory 类型**：
- **短期记忆**：对话历史（Context Window）
- **长期记忆**：向量数据库存储的事实和经验
- **实体记忆**：跟踪对话中提到的关键实体

**常见 Tools**：
- 搜索（Google、Bing、DuckDuckGo）
- 计算器（Code Interpreter）
- 数据库查询
- API 调用

## 关键决策树

### 什么时候用 RAG？
```
需要回答关于私有数据的问题？
    │
    ├── 是 → 使用 RAG
    │         │
    │         ├── 数据经常更新？
    │         │      ├── 是 → 实时索引更新
    │         │      └── 否 → 批量索引
    │         │
    │         └── 需要精确回答？
    │                ├── 是 → 小分块 + 重排序
    │                └── 否 → 大分块 + 直接生成
    │
    └── 否 → 直接用 LLM
              │
              └── 需要最新信息？
                     ├── 是 → LLM + 搜索工具
                     └── 否 → 纯 LLM
```

### 什么时候微调？
```
需要改变模型行为/风格？
    │
    ├── 是 → 考虑微调
    │         │
    │         ├── 数据量 > 1000 条？
    │         │      ├── 是 → LoRA 微调
    │         │      └── 否 → Prompt Engineering
    │         │
    │         └── 计算资源充足？
    │                ├── 是 → 全参数微调
    │                └── 否 → LoRA/QLoRA
    │
    └── 否 → Prompt Engineering + RAG
```

## 常见误区澄清

| 误区 | 真相 |
|------|------|
| "大模型什么都知道" | 有知识截止日期，会过时；训练数据有偏差 |
| "大模型不会犯错" | 会幻觉（编造事实），会犯错，需要验证 |
| "更大的模型一定更好" | 要看场景，小模型+好策略可能更优 |
| "微调能解决一切" | 微调成本高，先尝试 Prompt 和 RAG |
| "Prompt 越长越好" | 上下文有长度限制，且模型容易"遗忘"前面的内容 |
| "Temperature=0 最准确" | 对某些任务适度随机性反而更好 |
