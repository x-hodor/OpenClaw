## 一、基础题（了解概念）

### Q1: 什么是 Token？为什么模型不直接处理文字？

**参考答案**：
- Token 是模型处理文本的最小单位
- 原因：(1) 统一不同语言的表示 (2) 控制词表大小 (3) 处理未登录词
- 1 个汉字 ≈ 1.5 tokens，1 个英文单词 ≈ 1.3 tokens

**对应知识点**：Tokenization、BPE 算法

---

### Q2: 解释一下 Temperature 参数的作用

**参考答案**：
- 控制模型输出的随机性
- Temperature=0：确定性强，总是选概率最高的词
- Temperature=1：按概率分布采样，更多样
- 代码生成用低温度(0.2)，创意写作用高温度(0.8-1.0)

**对应知识点**：采样策略、Softmax 温度调节

---

### Q3: 什么是幻觉（Hallucination）？如何缓解？

**参考答案**：
- 定义：模型生成看似合理但实际错误的内容
- 类型：事实性幻觉（与事实不符）、忠实性幻觉（与输入矛盾）
- 缓解方法：(1) RAG 提供可靠信源 (2) 提示工程约束 (3) 后处理验证 (4) 选择合适的模型

**对应知识点**：模型局限性、RAG、Prompt Engineering

---

### Q4: 什么是 RAG？为什么需要它？

**参考答案**：
- RAG = Retrieval-Augmented Generation（检索增强生成）
- 流程：用户提问 → 检索相关文档 → 将文档注入 Prompt → LLM 生成回答
- 为什么需要：(1) 解决知识截止问题 (2) 利用私有数据 (3) 减少幻觉

**对应知识点**：RAG 架构、Embedding、向量检索

## 二、进阶题（理解原理）

### Q5: 解释 Transformer 中的 Self-Attention 机制

**参考答案**：
```
核心思想：每个词都能"看到"其他所有词，根据相关性加权聚合信息

计算过程：
1. 输入 X 通过线性变换得到 Q、K、V
2. Attention(Q,K,V) = softmax(QK^T/√d_k)·V
3. 除以 √d_k 是为了防止点积过大导致梯度消失
4. 输出是每个位置的上下文表示

复杂度：O(n²·d)，n 是序列长度，d 是模型维度
```

**对应知识点**：Self-Attention、Query/Key/Value、计算复杂度

---

### Q6: 为什么 Transformer 比 RNN 好？

**参考答案**：
```
对比维度：

1. 并行性
   RNN：串行计算，时间复杂度 O(n)
   Transformer：矩阵运算，可并行，时间复杂度 O(1) 顺序操作

2. 长距离依赖
   RNN：远距离信息需经过多步传递，容易遗忘
   Transformer：任意位置直接交互，路径长度 O(1)

3. 训练稳定性
   RNN：梯度消失/爆炸问题
   Transformer：残差连接 + LayerNorm，训练更稳定

代价：Attention 复杂度 O(n²)，长序列时计算量大
```

**对应知识点**：Transformer vs RNN、长距离依赖、并行计算

---

### Q7: 解释一下 Scaling Law

**参考答案**：
```
定义：模型性能与规模（参数、数据、算力）呈幂律关系

公式：L(N) = (N_c/N)^α
- L：损失，N：参数量
- 性能随规模平滑提升，可预测

关键发现：
1. 涌现能力：规模达到临界点，新能力突然出现
2. Chinchilla 最优：参数量 ∝ 数据量，不能只看参数
3. 预测性：可以用小模型预测大模型性能

实践意义：可以规划训练预算，预测模型能力
```

**对应知识点**：Scaling Law、涌现能力、Chinchilla

### Q8: LLM 的训练分为哪几个阶段？各阶段的作用是什么？

**参考答案**：
```
Stage 1: 预训练 (Pre-training)
- 数据：万亿级 tokens
- 任务：Next Token Prediction
- 产出：基础模型，有语言能力和世界知识
- 成本：最高（数百万美元）

Stage 2: 监督微调 (SFT)
- 数据：数十万高质量指令-回答对
- 任务：监督学习，学习对话格式
- 产出：指令模型，能遵循指令
- 成本：中等（数千美元）

Stage 3: RLHF
- 数据：人类偏好排序
- 任务：奖励模型 + PPO 强化学习
- 产出：对齐模型，安全、有用、诚实
- 成本：中等（数万美元）
```

**对应知识点**：训练流程、Pre-training、SFT、RLHF

## 三、高级题（深入理解）

### Q9: 解释一下 LoRA 的原理，为什么它比全参数微调好？

**参考答案**：
```
原理：
- 冻结原始参数 W₀
- 引入低秩矩阵 A、B，使得 ΔW = A×B
- 训练时只更新 A、B，参数量大幅减少
- 推理时 W = W₀ + A×B，不增加延迟

优势：
1. 参数效率高：可训练参数减少 99%+（如 7B 模型只需训练 10M 参数）
2. 不损失预训练知识：W₀ 保持不变
3. 多任务切换：不同任务训练不同的 A、B 适配器
4. 存储友好：每个适配器只有几十 MB

秩(rank)选择：通常 8-64，越大表达能力越强，但训练成本越高
```

**对应知识点**：LoRA、参数高效微调、低秩近似

---

### Q10: 为什么大模型会出现涌现能力？

**参考答案**：
```
涌现定义：模型规模达到临界点时，某些能力突然出现或大幅提升

可能原因：
1. 任务复杂度的阈值效应
   - 小模型：只能做简单的模式匹配
   - 大模型：有能力组合多个技能完成复杂任务

2. 高维空间的几何特性
   - 足够大的模型可以学习到数据的流形结构
   - 某些能力需要模型容量达到阈值才能"拟合"

3. 评估指标的离散性
   - 某些任务准确率从 0 到 1 是跳跃的
   - 实际上是连续提升，但指标看起来是"涌现"

争议：
- 有人认为涌现是评估方式造成的假象
- 但不可否认，大模型确实展现出了小模型不具备的能力组合
```

**对应知识点**：涌现能力、模型容量、复杂系统

### Q11: 如何评估 RAG 系统的效果？

**参考答案**：
```
检索阶段评估：
- Recall@K：正确答案在 Top-K 中的比例
- MRR (Mean Reciprocal Rank)：正确答案排名的倒数均值
- NDCG：考虑相关度等级的排序质量

生成阶段评估：
- 忠实度 (Faithfulness)：回答是否基于检索内容
- 相关性 (Relevance)：回答是否切题
- 用 LLM 作为评判者（GPT-4 打分）

端到端评估：
- 准备测试集（问题+期望答案）
- 人工评估或自动指标（ROUGE、BLEU）
- 对比实验（有 RAG vs 无 RAG）

常用工具：RAGAS、TruLens
```

**对应知识点**：RAG 评估、检索指标、生成指标

### Q12: 设计一个高并发的 LLM 服务，你会考虑哪些因素？

**参考答案**：
```
1. 推理优化
   - 量化（INT8/INT4）减少显存
   - KV Cache 避免重复计算
   - Continuous Batching 提高吞吐
   - 投机采样降低延迟

2. 架构设计
   - 负载均衡（多实例）
   - 请求队列 + 优先级调度
   - 流式响应（提升用户体验）
   - 断路器（防止级联故障）

3. 成本控制
   - 缓存常见查询结果
   - 模型路由（简单任务用小模型）
   - 动态扩缩容

4. 可观测性
   - 延迟、吞吐量、错误率监控
   - Token 消耗追踪
   - 模型漂移检测

技术选型：vLLM / TGI / TensorRT-LLM
```

**对应知识点**：推理优化、系统架构、生产部署

## 四、系统设计题

### Q13: 设计一个企业知识库问答系统

**参考答案框架**：
```
1. 数据处理层
   - 支持多格式文档（PDF/Word/网页）
   - 文档解析和清洗
   - 分块策略（按段落/语义/固定大小）

2. 索引层
   - Embedding 模型选择（中文用 BGE）
   - 向量数据库（Milvus/Pinecone）
   - 元数据存储（文档来源、更新时间）

3. 检索层
   - 混合检索（向量 + 关键词）
   - 重排序（Rerank）
   - 查询改写（Query Expansion）

4. 生成层
   - Prompt 工程（约束回答格式）
   - 引用溯源（显示参考文档）
   - 幻觉检测与过滤

5. 应用层
   - Web 界面
   - API 接口
   - 反馈收集（用于持续优化）
```

---

### Q14: 如何为特定领域（如医疗）定制 LLM？

**参考答案**：
```
1. 数据准备
   - 收集领域文档（论文、指南、病历）
   - 构建指令数据集（问题-回答对）
   - 数据清洗（去隐私、去错误）

2. 模型选择
   - 基础模型：通用 LLM 或领域预训练模型（如 Huatuo）
   - 决策：继续预训练 vs 直接微调

3. 训练策略
   - 继续预训练：在领域语料上进一步训练
   - 监督微调：SFT 在指令数据上
   - RLHF：专家反馈对齐

4. 安全保障
   - 事实性核查（不能替代医生）
   - 敏感词过滤
   - 免责声明

5. 评估
   - 领域专家评估
   - 构建领域测试集
   - 与通用模型对比
```

## 五、开放性问题

### Q15: 你认为 LLM 的下一个突破点在哪里？

**答题思路**：
- 多模态统一（文本+图像+音频+视频）
- 长上下文（百万级 token，能处理整本书）
- Agent 能力（自主规划、工具使用、长期记忆）
- 推理能力（数学、代码、逻辑）
- 效率优化（更小模型、更快推理、更低成本）
- 可靠性和安全性（减少幻觉、对齐人类价值观）

**加分项**：结合具体论文或产品（如 Gemini 1.5 Pro 的长上下文、Claude 的 Computer Use）

### Q16: 如何看待 LLM 的"理解"能力？它真的理解语言吗？

**答题思路**：
- 行为层面：能完成理解相关的任务（问答、推理、总结）
- 机制层面：基于统计模式匹配，非人类式的"理解"
- 哲学层面：图灵测试视角 vs 中文房间论证
- 实用层面：不管是否"真理解"，有用就是价值

**加分项**：提及不同观点（Stochastic Parrots vs World Models）
