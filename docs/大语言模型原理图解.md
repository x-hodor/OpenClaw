## 一、LLM 是什么？一句话解释

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   大语言模型 = 超级厉害的"文字接龙"玩家                      │
│                                                             │
│   你说前半句 → 它猜后半句 → 再猜下一句 → 一直猜下去         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 举个栗子 🌰

```
输入：今天天气很好，我想去
预测：公园

输入：今天天气很好，我想去公园
预测：散步

输入：今天天气很好，我想去公园散步
预测：，你觉得

...一直预测下去，就生成了一段话
```

**神奇之处**：当"接龙"玩得足够好，它就学会了写诗、编程、解数学题、回答问题...

## 三、核心技术：Transformer 架构

### 3.1 整体架构图

```
输入: "猫坐在垫子上"
         │
         ▼
┌─────────────────────────────────────────┐
│  1. 词嵌入 (Embedding)                  │
│     每个词 → 变成一个向量                │
└─────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────┐
│  2. 位置编码 (Position Encoding)        │
│     告诉模型: 这个词在句子的第几个位置   │
└─────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────┐
│  3. 多层 Transformer Block              │
│     核心处理层（6-96层不等）             │
└─────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────┐
│  4. 输出层                              │
│     预测下一个词是什么                   │
└─────────────────────────────────────────┘
         │
         ▼
输出: "它" (概率最高的下一个词)
```

### 3.2 核心：注意力机制（Attention）

#### 什么是注意力？

```
当模型读到这句话时：

"猫坐在垫子上，因为它很温暖"
                 ↑
            这里的"它"指谁？

人类理解：回头看，"它"指的是"垫子"
模型理解：计算每个词与"它"的相关性

┌─────────────────────────────────────────┐
│  注意力计算（示意图）                    │
│                                         │
│  "它" 与各个词的相关程度：              │
│                                         │
│  猫     [0.1]  ████                     │
│  坐在   [0.1]  ████                     │
│  垫子   [0.7]  ████████████████         │
│  上     [0.05] ██                       │
│  ，     [0.02] █                        │
│  因为   [0.02] █                        │
│  它     [0.0]                           │
│  很     [0.01]                          │
│                                         │
│  结论："它"最可能指"垫子"               │
└─────────────────────────────────────────┘
```

#### 自注意力计算流程

```
输入: 三个词 [我, 喜欢, 学习]

Step 1: 每个词生成三个向量
┌─────────┐
│   我    │ → Q(查询), K(键), V(值)
├─────────┤
│  喜欢   │ → Q(查询), K(键), V(值)
├─────────┤
│  学习   │ → Q(查询), K(键), V(值)
└─────────┘

Step 2: 计算注意力分数（谁关注谁）
            我    喜欢   学习
我        [0.5    0.3    0.2  ]
喜欢      [0.4    0.4    0.2  ]
学习      [0.2    0.3    0.5  ]

Step 3: 加权求和，得到新的词表示
"学习"的新表示 = 0.2×"我" + 0.3×"喜欢" + 0.5×"学习"
```

### 3.3 多头注意力（Multi-Head）

```
为什么要多头？
因为一句话中可能有多种关系：

┌─────────────────────────────────────────┐
│  "猫坐在垫子上"                         │
│                                         │
│  Head 1 (主谓关系):                     │
│  猫 ──→ 坐                              │
│                                         │
│  Head 2 (介词关系):                     │
│  在 ──→ 垫子                            │
│                                         │
│  Head 3 (指代关系):                     │
│  垫子 ←── 它                            │
└─────────────────────────────────────────┘

多个 Head 从多个角度理解句子，再合并结果
```

## 四、训练过程：三步走

### 4.1 整体流程图

```
┌─────────────────────────────────────────────────────────────┐
│                      LLM 训练三部曲                          │
└─────────────────────────────────────────────────────────────┘

    Step 1: 预训练 (Pre-training)
    ┌─────────────────────────────────────┐
    │                                     │
    │  海量互联网文本                      │
    │       ↓                             │
    │  自监督学习                          │
    │       ↓                             │
    │  预测下一个词                        │
    │       ↓                             │
    │  得到"基础模型"                     │
    │  (如 GPT-3, Llama)                  │
    │                                     │
    └─────────────────────────────────────┘
                    │
                    ▼
    Step 2: 监督微调 (SFT)
    ┌─────────────────────────────────────┐
    │                                     │
    │  高质量对话数据                      │
    │  (问题→答案格式)                    │
    │       ↓                             │
    │  有监督训练                          │
    │       ↓                             │
    │  得到"指令模型"                     │
    │  (如 ChatGPT, Claude)               │
    │                                     │
    └─────────────────────────────────────┘
                    │
                    ▼
    Step 3: 人类反馈强化学习 (RLHF)
    ┌─────────────────────────────────────┐
    │                                     │
    │  人类对多个回答打分                  │
    │       ↓                             │
    │  训练奖励模型                        │
    │       ↓                             │
    │  强化学习优化                        │
    │       ↓                             │
    │  得到"对齐模型"                     │
    │  (更符合人类价值观)                 │
    │                                     │
    └─────────────────────────────────────┘
```

### 4.2 预训练：阅读海量文本

```
训练数据举例：

文本: "爱因斯坦出生于德国"
     ↑
输入: "爱因斯坦出生于"
目标: "德国"

文本: "The cat sat on the mat"
     ↑
输入: "The cat sat on the"
目标: "mat"

... 重复万亿次

结果：模型学会了语言规律、世界知识、推理模式
```

### 4.3 监督微调：学会对话

```
微调数据格式：

┌─────────────────────────────────────┐
│  问题: 什么是机器学习？              │
│  答案: 机器学习是让计算机从数据中... │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  问题: 把这句话翻译成英文            │
│       "你好，世界"                   │
│  答案: "Hello, world"               │
└─────────────────────────────────────┘

... 数百万组高质量对话
```

### 4.4 RLHF：对齐人类偏好

```
为什么需要 RLHF？

情况：模型生成了 3 个回答
┌─────────────────────────────────────┐
│  回答 A: 正确但复杂难懂              │
│  回答 B: 正确且清晰易懂 ★            │
│  回答 C: 错误但有说服力              │
└─────────────────────────────────────┘

人类打分: B > A > C

RLHF 让模型学会：不仅要"对"，还要"让人满意"
```

## 六、关键技术概念图解

### 6.1 Token（词元）

```
文本: "Hello world"
       ↓ 分词
Tokens: ["Hello", " world"]
        或
Tokens: ["He", "ll", "o", " w", "or", "ld"] (子词)

类比：Token 是模型的"字母"
中文："机器学习" → ["机器", "学习"] 或 ["机", "器", "学", "习"]

GPT-4 上下文：约 128K tokens ≈ 10 万字
```

### 6.2 Embedding（嵌入）

```
把词变成数字向量，让计算机能处理：

"国王"   → [0.2, -0.5, 0.8, 0.1, ...]  (几百维)
"女王"   → [0.3, -0.4, 0.7, 0.2, ...]
"男人"   → [0.1, -0.6, 0.3, 0.0, ...]
"女人"   → [0.2, -0.5, 0.2, 0.1, ...]

有趣的性质：
国王 - 男人 + 女人 ≈ 女王
（向量能捕捉语义关系）
```

### 6.3 温度参数（Temperature）

```
控制输出的"随机性"

低温 (0.2): 保守、确定
"今天天气很好，我想去___"
→ "公园散步" (90% 概率)
→ 适合：问答、代码

高温 (1.0): 创造性、多样
"今天天气很好，我想去___"
→ "公园" (40%) / "海边" (35%) / "爬山" (25%)
→ 适合：创意写作、头脑风暴
```

### 6.4 思维链（Chain of Thought）

```
普通提问：
Q: 一个农场有鸡和兔，头共 35 个，脚共 94 只，各多少？
A: 鸡有 23 只，兔有 12 只。  ← 直接给答案，可能错

思维链提示：
Q: 一个农场有鸡和兔...
A: 让我一步步思考：
   1. 假设全是鸡，35×2=70 只脚
   2. 实际多 94-70=24 只脚
   3. 每只兔多 2 只脚，24÷2=12 只兔
   4. 鸡有 35-12=23 只
   所以鸡有 23 只，兔有 12 只。 ← 正确！

关键：让模型"一步步想"，推理能力大幅提升
```

## 十、核心概念速查表

| 概念 | 一句话解释 |
|------|------------|
| **Transformer** | LLM 的核心架构，用注意力机制处理序列 |
| **Attention** | 让模型关注输入中重要的部分 |
| **Token** | 模型处理的最小文本单位 |
| **Embedding** | 把文本变成计算机能理解的数字向量 |
| **预训练** | 在大规模数据上学习语言规律 |
| **微调** | 在特定任务上调整模型 |
| **RLHF** | 用人类反馈优化模型输出 |
| **Prompt** | 给模型的输入指令 |
| **Temperature** | 控制输出的随机性 |
| **Context Window** | 模型能记住的上下文长度 |
| **Hallucination** | 模型"胡说八道"的现象 |
| **RAG** | 检索增强生成，结合外部知识 |
